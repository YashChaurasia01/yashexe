Overview

Search by image.
Not by guessing.

DeepSearch is a reverse image search engine built on Content-Based Image Retrieval (CBIR) — the same core idea behind Google Lens. Upload any image and the engine extracts its visual features, then searches a pre-built index to surface the closest visual matches within the trained datasets, no keywords required.

The machine learning model — feature extractors, classifiers, and Annoy similarity indexes for every category — was built by Utkarsh Saxena ↗. The interface, experience, and live web application you're browsing were designed and developed here.

Note: DeepSearch is trained and indexed only on the six categories shown above — Animal, Fast Food, Flower, Fruit, Landscape, and Shoes. Search results are derived exclusively from these datasets; images outside these categories won't return meaningful matches.

Process

How we made it

01 Feature Extraction & Model

Built by Utkarsh Saxena ↗ — a feature extractor and classifier trained per category, with an Annoy index for fast nearest-neighbour similarity search.

02 Pipeline Assembly

Combined the extractor, classifier, and index into one working CBIR pipeline per dataset — functioning as a self-contained image search engine.

03 Web Interface

Designed and built the upload-and-search UI, connecting it to the trained models so results render instantly in the browser.

04 Deployment

Shipped the full application live, so anyone can try the model directly — no setup required.
